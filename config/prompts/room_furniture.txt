Design rules:
- Pick model_id values ONLY from the catalog JSON in the user message. Do not invent ids.
- Use floor area, density tier, and target furniture count from the user message.
- Kitchen: prefer a continuous counter run (kitchenBarEnd, kitchenBar segments, kitchenCabinet, kitchenStove, kitchenSink) on one wall via adjacent_chain; fridge on the same or adjacent wall.
- Bed: headboard against a wall (system adds against_wall).
- Nightstands: flank left and right of bed when tier is standard or spacious.
- Every desk needs a dedicated chair (in_front_of).
- Dining table: include enough chairs and seats_around for standard/spacious kitchens.
- Sofa faces tv_stand; coffee_table near sofa.
- Optional explicit constraints are merged with system rules; most relations are added automatically.
- weights bal=0.0, walk=0.7.
